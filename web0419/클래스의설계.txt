1. 시나리오 분석하기

우선은 클래스를 간단하게 나눈다.
그리고 나서 나누어져 있는 것들을 파악한다.

한글로 작성한 클래스

회원 클래스{
    이름: - 이름은 성과 이름으로 나눕니다.
    주소: 
    메일 주소:
    패스워드:
}

분석하여 다시 클래스를 추출()

회원 클래스{
    회원번호: - 연결
    성:
    이름:
    주소: - 주소는 별도의 클래스로 나눕니다.
    메일 주소:
    패스워드:
}
주소 클래스{
    회원번호: - 연결
    우편번호:
    시/도:
    시/군/구:
    읍/면:
    도로명 건물번호
    동/층/호
    (법정동/공동주택명)
    전화번호
}

회원 클래스에 회원번호를 추가하고 주소 클래스에도 같은 회원번호를 추가하면 두 클래스를 연결합니다.

2. 처리 추가하기

데이터를 추가했으면 필요한 기능들을 생각하여 써냈던 메소드들을 추가합니다.

회원 클래스{
    회원번호: - 연결
    성:
    이름:
    메일 주소:
    패스워드:
    회원 정보 등록(){}
    등록 완료 메일 송신(){}
    회원 정보 편집(){}
    패스워드 재발급(){}
    회원 탈퇴(){}
}

3. PHP의 문법으로 고쳐 쓰기

데이터는 클래스 내부에서만 처리하기 위해 접근제한 키워드인 private를 지정하여 
외부에서 접근할 수 없도록 합니다.
메소드는 모두 클래스 외부에서 실행하기 때문에 public으로 지정합니다.
